# 风险卡片格式（所有专家角色统一输出）

> 对齐 cce risk 风险卡片 schema。每条风险一张卡片，结构化、可统计、可汇总。
> 各专家角色（法务/财务/业务）按此格式产出卡片，chief-reviewer 汇总。

## 字段定义

| 字段 | 说明 | 取值 |
|------|------|------|
| 卡片编号 | rule_id | R01, R02, ... 按维度顺序编号 |
| 审查维度 | 本卡片审查的风险维度 | 如"知识产权归属"、"付款条件"、"排他条款" |
| 风险结果 | 是否存在风险 | 是 / 否 / 需人工检查 |
| 风险等级 | 严重程度 | 🔴高风险 / 🟡中风险 / 🟢低风险 |
| 风险分析 | 详细论证 | 100-300字，引用条款编号+原文+论证 |
| 涉及条款 | 合同原文 | 精确引用，不改写 |
| 修改建议 | 修改方案 | 类型(新增/修改/删除) + 修改前 + 修改后 |
| 法律依据 | 法条/准则/惯例 | 法务引法条、财务引准则、业务引惯例 |

## markdown 模板

每张卡片按此格式产出，所有卡片写入一个 artifact 文件上报：

```markdown
### 审查卡片: R01

| 字段 | 内容 |
|------|------|
| 审查维度 | 知识产权归属 |
| 风险结果 | 是 |
| 风险等级 | 🔴 |

**风险分析**
（100-300字论证，引用条款编号+原文）

**涉及条款**
- §2.3 "IP归甲方"（原文精确引用，不改写）

**修改建议**
- 类型: 修改
- 修改前: §2.3 知识产权归甲方所有
- 修改后: §2.3 背景知识产权归乙方所有；项目成果知识产权归甲方，但乙方保留为履行本合同所必需的永久免费使用权...

**法律依据**
- 《民法典》第八百六十四条（技术转让合同范围约定）
```

## 约束

- 每个审查维度必须产出一张卡片，不跳过
- 合同条款引用原文，不改写
- 不确定的标记为"需人工检查"
- 风险等级与风险结果对应：是→🔴/🟡，否→🟢，需人工检查→🟡
- 法律依据：法务引中国法条、财务引会计准则/财务规范、业务引商业惯例/行业实践
- 所有卡片写入一个 artifact 文件（如 `reviews/pi-legal-POOK_001-cards.md`），通过 report_role_result 上报

## chief-reviewer 汇总格式

chief-reviewer 收集各专家卡片后，输出汇总报告：

```markdown
## 合同审查风险卡片汇总

### 风险统计
- 总卡片数：N 张
- 🔴 高风险：X 张 | 🟡 中风险：Y 张 | 🟢 低风险：Z 张 | 需人工检查：W 张
- 按角色：法务 A 张 | 财务 B 张 | 业务 C 张

### 4a 交叉风险卡片（三方各自 OK，合在一起出问题）
[按上述卡片格式产出交叉风险卡片]

### 4b 视角冲突（同条款立场相反，并列不裁决）
- [条款 X.X] 法务立场：<X> | 财务立场：<Y> | 业务立场：<Z> | 需你裁决

### 4c 遗漏补位
- <遗漏维度> | 建议补充 | 类型：真遗漏/本合同未涉及

### 风险优先级视图（P0/P1/P2，非决策建议）
- P0（必须修改）：R01 <风险> | R03 <风险>
- P1（建议修改）：...
- P2（知悉即可）：...

### ⚠️ 本报告不出可签/不可签裁决
```
